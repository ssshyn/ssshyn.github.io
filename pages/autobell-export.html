2024.04 — 2024.09 Java8 Spring Boot Tibero JPA QueryDSL Swagger

프로젝트 개요

현대글로비스의 중고차 수출 플랫폼 운영 효율성 향상을 위해, 주문부터 선적, 수령까지 단계별로 분리되어 있던 프로세스를 통합 관리할 수 있도록 플랫폼 전반을 고도화하는 프로젝트에 참여하였습니다.

기존에는 각 단계가 개별 메뉴로 운영되어 데이터 흐름이 단절되고 관리 효율이 떨어지는 문제가 있었으며,
이를 개선하기 위해 업무 프로세스를 재정의하고 통합된 관리 구조로 리뉴얼하는 것이 주요 목적이었습니다.

본 프로젝트는 기획 3명, FE 5명, BE 5명이 참여한 다수의 기능 고도화와 신규 개발이 포함된 프로젝트로,
저는 백엔드 개발자로 참여하여 선적 프로세스 전반 리뉴얼 및 인천수출센터 관리 기능 개발을 담당하였습니다.

특히, 선박-차량 매핑 로직을 개선하여 선적 관리의 정확도를 높였으며, 차량 상태 관리 및 협력사 평가 기능을 고도화하여 운영 데이터의 신뢰도를 향상시켰습니다.

또한, 인천수출센터에서 수기로 처리되던 업무를 디지털화하기 위해 QR 기반 기능을 도입하고 PC 및 모바일 환경에서 활용 가능한 관리 메뉴를 구현함으로써 현장 업무 효율성을 개선하는 데 기여하였습니다.

주요 업무

  • 중고차 수출, 선적, 수령 프로세스 시스템 고도화
  • 선적 도메인 기능 개선 API 개발
  • QueryDSL 기반 동적 쿼리를 적용하여 다양한 검색 조건을 처리할 수 있도록 조회 로직 개선
  • Oracle 기반 Tibero DB에서 프로시저를 활용한 데이터 마이그레이션 로직 구현

기술 스택

Java8 Spring Boot Tibero JPA QueryDSL Swagger

트러블슈팅

다중 도메인 통합 조회 응답 지연
Problem

차량, 주문, 선적, 입출고 상태 등 여러 도메인 정보를 통합 조회하는 기능에서 다수 테이블 조인과 복잡한 상태 조건이 포함되면서 조회 응답 시간이 약 12초 이상 소요되는 문제가 발생하였습니다. 이로 인해 사용자 화면에서 데이터 로딩 지연이 발생하고, 서비스 사용성이 저하되는 이슈가 있었습니다.

Cause

단일 차량/주문 데이터를 조회하기 위해 차량 정보, 제원 정보, 주문, 선적, 입출고 상태 등 여러 테이블을 조인하여 데이터를 조회하는 구조였으며, 특히 검색 조건에 따라 동적으로 쿼리가 생성되면서 조회 대상 컬럼에 대한 인덱스가 부족하여 불필요한 Full Scan이 발생하는 것이 주요 원인이었습니다.

Solve

조회 성능 개선을 위해 주요 검색 조건 및 조인에 사용되는 컬럼을 분석하고, 해당 컬럼에 인덱스를 추가하여 쿼리 실행 계획을 개선하였습니다. 그 결과, 불필요한 Full Scan을 줄이고 조회 성능을 개선하여 응답 시간을 약 12초에서 8초 수준으로 단축할 수 있었습니다.

회고

해당 프로젝트를 통해 기존에 경험하지 못했던 DB 중심의 데이터 처리 방식을 다루며, 쿼리 구조와 성능을 고려한 백엔드 개발의 중요성을 체감할 수 있었습니다. 특히 다수 테이블 조인과 상태 조건을 포함한 복잡한 조회 로직을 개선하는 과정에서 데이터 구조와 인덱스 설계가 서비스 성능에 미치는 영향을 직접 경험할 수 있었습니다.

또한, 프로젝트 결과물에 대해 현업 사용자로부터 긍정적인 피드백을 받은 경험은 개발 결과가 실제 비즈니스 성과로 이어질 수 있다는 점을 체감할 수 있었던 의미 있는 경험이었습니다. 이를 통해 단순 기능 구현을 넘어, 사용자 관점에서의 개선이 중요하다는 점을 인식하게 되었습니다.

다만, 기존 시스템을 전면 개편하지 않고 기능 고도화를 지속적으로 진행하는 구조이다 보니 중복 데이터 및 불필요한 컬럼, 복잡도가 증가한 서비스 로직 등 유지보수성이 저하된 상태를 경험하였습니다.

이를 통해 기능 추가뿐만 아니라 구조 개선과 리팩토링의 필요성, 그리고 장기적인 관점에서의 시스템 설계가 중요하다는 점을 느끼게 되었으며, 향후에는 기능 개발과 함께 코드 품질 및 데이터 구조 개선을 함께 고려하는 개발을 지향하고 있습니다.
