삼성증권 투자스쿨 리뉴얼

애드캡슐소프트

2023.04 — 2023.09 Java8 Spring Boot MariaDB JPA JEUS8

프로젝트 개요

투자 교육 콘텐츠 접근성과 사용자 학습 경험 향상을 위해,
기존 투자 교육 영상 플랫폼을 리뉴얼하는 프로젝트에 참여하였습니다.

영상 시청 이력 트래킹을 기반으로 사용자의 학습 진행 상태를 관리하고,
레벨 테스트 기능을 통해 학습 성과를 점검할 수 있는 기능을 제공하는 서비스입니다.

본 프로젝트는 기획 1명, 디자이너 1명, FE 2명, BE 1명으로 구성된 소규모 팀으로 진행되었으며,
저는 백엔드 개발자로서 서비스 전반의 API 설계 및 데이터 처리 로직을 단독으로 담당하였습니다.

특히, 영상 시청 이력 기반의 상태 관리 및 학습 흐름을 고려한 데이터 구조를 설계하고,
FE와의 협업을 통해 사용자 경험을 반영한 API를 제공함으로써 학습 진행 상태를 안정적으로 관리할 수 있도록 기여하였습니다.

주요 업무

  • 기존 Spring MVC 프로젝트를 REST API 구조로 전환
  • 게시판 및 학습 콘텐츠 관리 API 설계 및 구현
  • SNS 로그인 연동 (OAuth2) 구현
  • 보안 취약점 (XSS, 세션 관리, 쿠키 설정 등) 개선 조치
  • KMCert 본인인증, 보안키패드 등 외부 솔루션 연동

기술 스택

Java8 Spring Boot MariaDB JPA JEUS8

트러블슈팅

SNS 로그인 API 키 환경 불일치 이슈
Problem

SNS 로그인 기능 적용 과정에서 운영/개발 환경에 동일한 API 키를 사용하도록 전달받아 적용하였으나,
운영 배포 후 환경별로 서로 다른 설정이 적용되어 인증이 정상 동작하지 않는 문제가 발생하였습니다.

Cause

API 키가 개인 계정 기반으로 관리되고 있어 환경별로 관리되는지 담당자 측에서도 확인이 어려웠고,
SNS 서버에서의 설정이 일관되지 않게 구성되어 있음을 확인하였습니다.

Solve

이에 따라 신규 API 키를 발급받아 운영 환경에 맞게 재설정하고,
환경별 설정을 명확히 구분하여 적용함으로써 인증 기능을 정상화하였습니다.

보안 점검 대응 및 취약점 개선
Problem

과거 개인정보 노출 이슈로 인해 보안 요구사항이 강화된 상태에서 프로젝트가 진행되었으며,
이에 따라 모의 해킹 및 보안 점검 대응을 수행하였습니다.

Solve

세션 탈취 가능성, XSS 공격, 쿠키 보안 설정, 개인정보 암호화 여부 등을 점검하고,
취약점이 발생할 수 있는 부분에 대해 보완 작업을 진행하였습니다.

이를 통해 서비스 보안 수준을 강화하고, 안정적으로 운영 가능한 환경을 구축하였습니다.

기획 변경에 따른 일정 리스크 대응 및 협업 개선
Problem

프로젝트 진행 중 기획 인력 이슈로 인해 요구사항 변경이 빈번하게 발생하고 일정 지연 리스크가 존재하였습니다.

Solve

이에 따라 프론트엔드 개발자와 협업하여 기획서를 직접 검토하고,
시스템 구현 가능 여부 및 개발 범위를 기준으로 요구사항을 재정의하였습니다.

그 결과, 개발 일정 내 구현 가능한 범위로 기능을 조정하고,
불확실한 요구사항을 정리하여 프로젝트를 안정적으로 마무리할 수 있었습니다.

회고

소규모 팀에서 백엔드 개발을 단독으로 담당하며, 서비스 전반의 API 설계와 데이터 처리 흐름을 주도적으로 경험할 수 있었습니다.


특히, 영상 시청 이력 트래킹 및 학습 상태 관리 기능을 구현하면서 단순 데이터 전달을 넘어서 사용자 흐름을 고려한 상태 관리의 중요성을 이해하게 되었습니다.


또한, 프로젝트 진행 중 기획 변경 및 일정 이슈가 발생하는 상황에서 프론트엔드와 협업하여 요구사항을 재정의하고,
개발 가능 범위를 기준으로 기능을 조정하는 경험을 통해 현실적인 서비스 개발에서의 우선순위 설정과 협업의 중요성을 체감하였습니다.


이를 통해 백엔드 개발자로서 단순 구현을 넘어서, 서비스 구조와 사용자 경험, 그리고 협업까지 고려하는 시각을 갖게 된 경험이었습니다.
